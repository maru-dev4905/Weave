기준 산출물: wv.css, common.css 작성 기준: 현재 실제 포함되는 클래스 우선

1. 전체 구조

파일 역할 설명
common.css 토큰 레이어 색상, inner/gutter, radius 같은 CSS 변수 정의
wv.css 유틸리티/기본 스타일 레이어 reset, a11y, layout, utility class, helper class, motion class 포함
admin.css 별도 관리자 스타일 wv.css와 별개 성격
gov.css 별도 관공서 스타일 wv.css와 별개 성격

2. common.css와 wv.css 관계

  1. common.css가 :root에 기본 토큰 값을 선언합니다.
  2. wv.css 유틸리티 클래스는 이 토큰을 var(--token) 형태로 참조합니다.
  3. 토큰 값을 바꾸면 유틸리티가 참조하는 실제 색상, 간격, 반경도 함께 변경됩니다.

핵심 개념: wv.css는 유틸리티의 형태를 제공하고, common.css는 유틸리티가 참조하는 값을 제공합니다.

토큰 파일 토큰 소비 클래스 예시 실제 효과
common.css --g_50 .fc_g_50 텍스트 색상이 바뀜
common.css --p_40 .bgc_p_40 배경색이 바뀜
common.css --rd_base .rd_base border-radius가 바뀜

5. 토큰 체계

5.1 Layout 토큰

토큰 그룹 현재 변수 설명
Inner width --inner_sm, --inner_base, --inner_lg, --inner_xl 컨테이너 최대 너비
Gutter --gutter_sm, --gutter_base, --gutter_lg, --gutter_xl 좌우 패딩
Radius --rd_xs, --rd_sm, --rd_md, --rd_lg, --rd_xl 의미 기반 반경 토큰

5.2 Color 토큰

토큰 그룹 접두사 의미
Gray g_ 회색 스케일
Primary p_ 메인 컬러
Danger d_ 에러/위험
Warning w_ 경고
Success su_ 성공

7. Layout / Object

클래스 의미 실제 스타일
.inner 기본 컨테이너 max-width: var(--inner_base)와 gutter 적용
.inner_sm ~ .inner_xl 사이즈별 컨테이너 max-width: var(--inner_*)
.gutter_* 좌우 padding padding-left/right: var(--gutter_*)
.w_max_content Width Max Content width: max-content !important

8. Display / Flex / Gap

패턴 의미
.dp_b, .dp_f, .dp_g display block / flex / grid
.fd_col, .fd_row flex direction 제어
.al_center, .jtf_bet align-items / justify-content
.gap4 ~ .gap300 interval scale 기반 gap

9. Grid / Spacing / Typography

분류 예시 의미
Grid .grid_lay, .colST1, .colED12 12열 기반 배치
Spacing .mt_20, .pb_40, .md_pt_10 마진, 패딩, 반응형 간격
Typography .fs_16, .fw_bold, .ta_center 글자 크기, 굵기, 정렬
Radius .rd_base, .rd_full 반경 토큰과 pill 형태

주의: show 계열 클래스는 대부분 display: block !important로 동작하므로 원래 Flex나 Grid였던 요소를 완전히 복원하지는 않습니다.

11. 문서 요약

  • common.css는 토큰 저장소, wv.css는 유틸리티 레이어입니다.
  • 색상, 간격, 사이즈는 모두 토큰 기반으로 연결됩니다.
  • 반응형은 tb_, md_ 접두사와 res_* 클래스로 제어합니다.
  • 실제 수정이 많아질 때는 SCSS 원본과 빌드 결과를 함께 확인하는 편이 안전합니다.
